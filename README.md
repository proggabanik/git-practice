## Conclusion

In this lab, we gained hands-on experience with essential software engineering tools used for project management, collaboration, and version control. We explored **Trello** as a project management platform, where we created boards, lists, and cards to organize tasks, assign responsibilities, monitor project progress, and manage deadlines efficiently within a team.

We also learned the basic concepts and practical usage of **Git** a distributed version control system widely used in software development. During the lab we performed key Git operations such as initializing a repository, configuring user information, checking repository status, staging files, committing changes with meaningful commit messages and pushing local repositories to a remote GitHub repository. We also understood how Git maintains version history, enables collaboration among multiple developers and simplifies tracking and managing changes in a project.

Overall, this lab provided practical exposure to industry-standard tools that support project planning, team coordination, and source code management. These tools play a vital role in modern software development by improving collaboration, maintaining project organization, and ensuring efficient version control throughout the software development lifecycle.
